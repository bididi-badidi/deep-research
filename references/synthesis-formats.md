# Synthesis Formats

Templates for the final deliverable. Select based on the output format specified in
the Research Brief.

---

## Format A — Executive Summary (300–600 words)

Use when: audience is senior decision-makers; they need the bottom line fast.

```
EXECUTIVE SUMMARY
=================
Topic            : [primary research question, plain language]
Prepared for     : [audience]
Date             : [date]
Confidence level : [overall — based on weakest task]

BOTTOM LINE
-----------
[2–3 sentences directly answering the primary question. No hedging.]

KEY FINDINGS
------------
1. [Finding — one sentence. Source: citation]
2. [Finding — one sentence. Source: citation]
3. [Finding — one sentence. Source: citation]
(max 5 findings)

CAVEATS
-------
[Any significant gaps, conflicting evidence, or data quality issues]

RECOMMENDED NEXT STEPS
-----------------------
[1–3 concrete actions or follow-up research questions]

SOURCES
-------
[Abbreviated list — full citations in appendix]
```

## Format B — Full Research Report (1,000–3,000 words)

Use when: audience is researchers, analysts, or technical stakeholders who need depth.

```
RESEARCH REPORT
===============
Title            : [Descriptive title including the research question]
Brief ID         : [from Research Brief]
Date             : [date]
Audience         : [specified audience]

1. EXECUTIVE SUMMARY
[150–200 word summary of the full report]

2. INTRODUCTION
[Background and context. Why this question matters. Scope and limitations stated
upfront.]

3. METHODOLOGY NOTE
[Brief description of how the research was conducted — sources consulted, time
period, geographic scope, source exclusions applied]

4. FINDINGS
[Organised by task category or logical flow. Each sub-section addresses one
aspect of the primary question. Every claim is cited.]

  4.1 [Sub-heading matching a task category]
  4.2 [Sub-heading]
  4.3 ...

5. COMPARATIVE ANALYSIS (if applicable)
[Present side-by-side comparisons, contrasting findings, or conflicting evidence]

6. DISCUSSION
[Interpret what the findings mean in aggregate. Surface any tensions between
sub-findings. Note what remains uncertain.]

7. CONCLUSION
[Direct answer to the primary research question. Forward-looking statement.]

8. GAPS AND LIMITATIONS
[Honest account of what could not be found, weak evidence areas, and scope
constraints]

REFERENCES
[Full citation list in the format requested]
```

## Format C — Bullet-Point Briefing (100–300 words)

Use when: audience needs rapid consumption; findings will be discussed in a meeting.

```
RESEARCH BRIEFING
=================
Question : [primary question]
Date     : [date]

WHAT WE KNOW
• [Finding — max 15 words. (Source)]
• [Finding]
• [Finding]
• [Finding]
• [Finding]

WHAT'S UNCERTAIN
• [Gap or conflicting finding]
• [Gap or conflicting finding]

BOTTOM LINE
[One sentence.]
```

## Format D — Comparative Table

Use when: the research question compares multiple options, regions, or time periods.

```
COMPARATIVE RESEARCH TABLE
==========================
Question : [primary question]
Date     : [date]

| Dimension       | Option A        | Option B        | Option C        |
|-----------------|-----------------|-----------------|-----------------|
| [Dimension 1]   | [Finding]       | [Finding]       | [Finding]       |
| [Dimension 2]   | [Finding]       | [Finding]       | [Finding]       |
| [Dimension 3]   | [Finding]       | [Finding]       | [Finding]       |
| Strength of ev. | [High/Med/Low]  | [High/Med/Low]  | [High/Med/Low]  |

NOTES
[Any cells where evidence was missing or conflicting]

SOURCES
[Citation list]
```

## Calibrating for Audience

| Audience type      | Vocabulary                          | Length             | Citations                        |
| ------------------ | ----------------------------------- | ------------------ | -------------------------------- |
| General public     | Plain language, no jargon           | Short              | Minimal (link-style)             |
| Business executive | Business terms, no technical jargon | Brief with visuals | Key sources only                 |
| Technical expert   | Domain terminology preserved        | Can be long        | Full academic citations          |
| Student            | Clear, didactic, with definitions   | Medium             | Standard academic format         |
| Policymaker        | Plain language, outcome-focused     | Medium             | Referenced with reliability note |
